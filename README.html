Strategic Trend Analysis Pipeline

This tool downloads Google Trends data for a semantic core of keywords (expanded from a base word like love), forecasts yearly seasonality patterns using Facebook Prophet, clusters similar patterns, and generates an interactive HTML report with zoomable daily charts, a weekly activity heatmap, and peak tables.

The pipeline is fully local, caches downloaded data as CSV files, and supports a --data-ready mode to reuse cached data without re‑downloading.

✨ Features

  • Semantic expansion – DataMuse + Google Trends suggestions.
  • Local CSV caching – data/{base_keyword}/{keyword}.csv.
  • Prophet forecasting – Additive yearly seasonality, normalised.
  • Peak detection – scipy.signal.find_peaks (prominence ≥ 0.1).
  • Optimal clustering – K‑Means with silhouette score.
  • Interactive HTML report – cluster plots, weekly heatmap, peak tables.
  • --data-ready – reuses cached data, no API calls.

📦 Requirements

  • Python 3.8+
  • Install: pip install pandas numpy scipy scikit-learn pytrends prophet requests urllib3 plotly
Note: On some systems, pystan==2.19.1.1 may be needed before prophet.

🛠 Installation & Usage

  1. Save the script as full_analysis.py.
  2. Install dependencies (see above).
  3. Run: 
    python full_analysis.py --base-keyword love
  4. Second run (use cached data): 
    python full_analysis.py --base-keyword love --data-ready

Output: Full_Analysis_love_2026.html

📊 Understanding the Report

  • Cluster plots – daily normalised curves, red triangles = peaks. Hover, zoom.
  • Weekly heatmap – average influence per weekday/week. Week ranges inside Monday column.
  • Keyword vs Month table – exact peak dates (DD/MM).
  • All predicted peak dates – chronological list.

🧠 How It Works

  1. Expands base keyword into semantic core.
  2. Downloads/caches Trends data.
  3. Trains Prophet (additive yearly seasonality) on all data.
  4. Extracts & normalises yearly component for forecast year.
  5. Detects peaks.
  6. Clusters curves (silhouette, 2–6 clusters).
  7. Generates HTML report with Plotly.

⚠️ Rate Limiting

Google Trends enforces strict limits. The script waits 40‑60 seconds between keywords. Use --data-ready after the first run to avoid further downloads.
